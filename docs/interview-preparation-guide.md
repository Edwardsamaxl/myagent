## 六、面试高频 Q&A 准备卡

### Fin-agent 篇

**Q1：请介绍一下 Fin-agent**
> "Fin-agent 是一个面向金融场景的 RAG + 多 Agent 问答系统。整体架构是 Coordinator 模式：用户问题进来后，Coordinator 分发到 Planner 拆解任务，然后 RAG Worker 和计算 Worker 协作执行，最后 Synthesizer 汇总结果生成答案。检索链路用的是混合检索（BM25 + TF-IDF + Embedding），Embedding 异常时会自动降级保证可用性，最后通过两阶段重排（规则初筛 + bge-reranker 精排）把 Top-3 送到生成阶段。"

**Q2：LangGraph 和 LangChain 的 ReAct Agent 有什么区别？**
> "我们用的是 LangGraph 的 `create_react_agent`（来自 `langgraph.prebuilt`），不是 LangChain 的。这是 LangGraph 内置的 ReAct 实现，底层用的是 `StateGraph` + `AgentState` 状态机。每个节点是一个 Python 函数，边定义了状态转换，`should_continue` 节点控制循环终止。LangChain 的 `create_react_agent` 是另一个封装，两者是独立的。LangGraph 的优势是状态机逻辑完全可控，适合复杂的多 Agent 编排场景。"

**Q3：混合检索权重怎么定的？**
> "我们的融合策略支持 RRF 和加权求和两种模式。RRF 模式下 k=60，不同检索方式平等融合；加权求和模式下，我们通过小规模测试集（N=XXX）对比了多组权重配置，最终选择了当前配比。Embedding 通道有健康检查机制，异常时自动降级，实际是退化为词频检索，保证服务可用。"

**Q4：Embedding 降级怎么实现的？**
> "我们在服务启动和每次检索前会用单句测试向量是否返回，如果请求超时或返回空，就判定为异常。降级后会把 embedding_weight 设为 0，仅使用 BM25 + TF-IDF 检索，同时通过 logging.warning 提示修复建议，用户感知到的是语义检索暂时不可用，但系统仍能返回结果。"

**Q5：多 Worker 协作是怎么实现的？**
> "Coordinator 负责整体调度。Planner 拆解后，同批次就绪的任务按依赖拓扑排序协作执行，有依赖关系的任务会等待前序完成后才执行。Worker 之间通过 `WorkerResult` 数据结构传递结果，Synthesizer 最后汇总所有 Worker 输出生成答案。任务粒度的并行执行是后续可优化的方向（如 ThreadPoolExecutor）。"

### YOLOv8 篇

**Q1：CBAM 和 SE、ECA 的区别是什么？**
> "SE（Squeeze-and-Excitation）只做通道注意力，通过全局池化 + FC + Sigmoid 生成通道权重。ECA（Efficient Channel Attention）用 1D 卷积替代 FC，减少参数的同时保留局部跨通道交互。CBAM 是两者的结合：先做通道注意力（CAM），再做了空间注意力（SAM）。通道注意力通过共享 MLP 学习通道权重；空间注意力通过 MaxPool 和 AvgPool 沿通道维度池化后拼接，再卷积生成空间权重图。"

**Q2：WIoU v3 是什么？和 IoU、GIoU、DIoU、CIoU 有什么关系？**
> "IoU 是交集除以并集；GIoU 引入了最小包围框解决不相交情况；DIoU 引入了中心点距离；CIoU 在 DIoU 基础上加了长宽比约束；WIoU 在此基础上加入了动态聚焦机制——当样本 IoU 高时降低权重减少对易样本的惩罚，当 IoU 低时提高权重让模型更关注难样本，通过一个动态的 scale factor 实现。"

**Q3：消融实验怎么设计的？**
> "我们做了三组对照实验：baseline（标准 YOLOv8m）、单独加 CBAM、单独加 WIoU v3、以及两者联合使用。每组实验用相同的训练配置（epoch=300、lr=0.01、batch=16），在 NEU-DET 验证集上评估 mAP@0.5。实验只跑了单次，没有多次求平均，这是后续需要补充的。"

**Q4：伪标签阈值怎么定的？**
> "我们用了自适应阈值策略：初始阈值 0.4，每轮 Self-training 后根据伪标签分布的置信度分位数调整（比如取前 50% 样本的置信度中位数作为下一轮阈值）。同时加了翻转一致性筛选：同一样本正反翻_augmentation 都生成伪标签，只有两者一致的才保留，减少噪声伪标签的影响。"
