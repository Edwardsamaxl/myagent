# Ingestion 模块说明

## Ingestion 在做什么？

Ingestion（文档摄入）负责把**原始文本**变成**可检索的块（chunk）**，供后面的 retrieval 使用。

整体流程：`原始文本 → 清洗 → 切块 → 去重 → 输出 DocumentChunk 列表`

推荐使用 Markdown 格式（`scripts/extract_pdf_to_md.py`）作为输入，可保留 ##/### 结构，分块更合理。

---

## 四步拆解

### 1. 清洗 `_clean_text`

| 操作 | 目的 |
|------|------|
| `\r\n` / `\r` 统一成 `\n` | 换行符标准化 |
| 连续空格/制表符压缩成单个空格 | 减少噪音 |
| 连续 3 个以上换行压缩成 2 个 | 去掉大片空白 |

**输入**：任意长度文本  
**输出**：规范化后的单段文本

---

### 2. 切块 `_split_text`（结构感知策略）

当文本含 `##` 或 `###`（Markdown 标题）时，采用**结构感知分块**：

1. **按 ##/### 切出小节**：每个 `## 第二节...` 或 `### 一、...` 视为边界
2. **小节过长**：按双换行（段落）再切
3. **仍过长**：按单换行切，**不在行中断开**
4. **重叠**：相邻块保留 `chunk_overlap` 字符重叠

无结构时回退为：固定长度 + 优先在换行处结束。

- **为什么优先换行？** 财报中多为「一行一条信息」，行中断开会破坏语义
- **为什么 500 / 80？** 500 字符约 150～200 中文；80 字符重叠便于衔接

---

### 3. 去重 `_dedupe_chunks`

- 对每个块计算 SHA1 哈希
- 相同内容的块只保留一份
- 年报里重复的“重要提示”“目录”等会被去掉

---

### 4. 输出 `DocumentChunk`

每个块包含：

| 字段 | 含义 |
|------|------|
| `chunk_id` | 如 `年报_2024-0042` |
| `doc_id` | 文档 ID（如 `年报_2024`） |
| `text` | 块内容 |
| `source` | 来源标识（如 `贵州茅台/年报_2024.txt`） |
| `metadata` | 额外信息（如 hash） |

---

## 与入库的关系

当前链路：

1. **提取**：`scripts/extract_pdf.py` 把 PDF → txt，放到 `data/raw/finance/贵州茅台/`
2. **入库**：通过 Web `/api/ingest` 或批量脚本，读取 txt 内容，调用 `ingestion.ingest_text()` 得到 chunks
3. **索引**：`RagAgentService.ingest_document()` 把 chunks 交给 `retriever.upsert_chunks()` 建立检索索引

所以：**ingestion 只负责“切好块”，不负责持久化**。持久化在 `retriever.upsert_chunks()` 里（当前是内存索引，重启会清空）。
