# 规范补充说明

仅在需要核对细节时阅读；日常以 `SKILL.md` 为准。

## 与「用户规则」对齐的要点

| 场景 | 约定 |
|------|------|
| 对话语言 | 简体中文 |
| 代码展示 | 新代码用普通 markdown 代码块；**已有仓库文件**用带行号与路径的引用块 |
| 文档范围 | 不主动新建用户未要求的 md；任务明确要求时再写 |
| 提交/PR 描述 | 完整句、相关细节、好语法 |

## 术语一致性

在文档与代码注释中，对同一概念固定用词（例如始终用「接口」或始终用「API」，不要混用多种说法）。

## 何时必须读本文件

- 用户要求「严格对齐某份规范」或出现歧义时
- Skill 正文不足以覆盖边界情况时
